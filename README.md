# Obsidian Demo 1

这是一个为 **PARA × Zettelkasten × 内容工厂** 设计的 Obsidian 仓库示例。

目标：
- 同时支持长期知识沉淀与高频内容生产
- 让人和 AI 下次进入仓库时都能快速上手
- 避免“资料、思想、项目、草稿、复盘”混在一起

## 快速开始

### 给人类
1. 先看 `00_System/START-HERE.md`
2. 再看 `00_System/WORKFLOW.md`
3. 想写内容就从 `05_Content/Inbox/` 或 `05_Content/Topic-Cards/` 开始
4. 想沉淀想法就从 `04_Knowledge/Fleeting/` 或 `04_Knowledge/Permanent/` 开始

### 给 AI / 助手
**每次新进入这个仓库时，优先按以下顺序阅读：**
1. `00_System/AI-OPERATOR.md`
2. `00_System/START-HERE.md`
3. `00_System/WORKFLOW.md`
4. `00_System/TAG-GLOSSARY.md`
5. 当前活跃项目：`01_Projects/`
6. 当前活跃领域：`02_Areas/`

## 核心设计原则
- **PARA 管上下文**：Project / Area / Resource / Archive
- **Zettelkasten 管认知**：Fleeting / Literature / Permanent / MOC
- **Content Pipeline 管产出**：Inbox / Topic / Angle / Draft / Published / Feedback / Repurposing

## 一句话理解
- 资料归资料
- 思想归思想
- 内容归内容
- 项目归项目

## 推荐习惯
- 不要把所有东西都叫“卡片”然后混放
- 一张卡尽量只承载一个核心点
- 所有已发布内容都要有复盘
- 好内容要进入 `Repurposing/`
- 重要观点优先沉淀成 `Permanent/`

## 已包含的完整示例
1. **观点驱动型**：全自动化内容工厂为什么容易失去人味
2. **用户问题驱动型**：为什么很多人做内容坚持不下来
