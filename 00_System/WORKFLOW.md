# WORKFLOW

## 总体流转

### 知识流（Zettelkasten）
`Fleeting → Literature → Permanent → MOC`

### 内容流（Content Pipeline）
`Inbox → Topic → Angle → Draft → Published → Feedback → Repurposing`

### 项目流（PARA / Projects）
`Project Overview → Goals → Selected Topics → Drafts → Publishing → Review`

## 三套系统如何协作

- `03_Resources/` 提供原材料
- `04_Knowledge/` 提供认知和长期观点
- `05_Content/` 把观点转成内容资产
- `01_Projects/` 在特定时间段组织一场内容战役
- `02_Areas/` 保存长期经营的主题框架

## 一条内容的标准路径
1. 原始问题 / 灵感进入 `05_Content/Inbox/`
2. 外部资料整理进 `04_Knowledge/Literature/`
3. 成熟判断沉淀为 `04_Knowledge/Permanent/`
4. 选题进入 `05_Content/Topic-Cards/`
5. 切角度进入 `05_Content/Angle-Cards/`
6. 写成草稿进入 `05_Content/Draft-Cards/`
7. 发布后进入 `05_Content/Published/`
8. 复盘进入 `05_Content/Feedback/`
9. 表现好则进入 `05_Content/Repurposing/`

## 什么时候用 PARA
- 一个东西与具体目标、期限、交付强相关时，用 `Project`
- 一个东西是长期经营方向时，用 `Area`
- 一个东西只是参考材料时，用 `Resource`
- 一个东西过期但可能未来还要回看时，用 `Archive`

## 什么时候用 ZK
- 当你需要把“别人说的”变成“你自己的认知”
- 当你希望观点能长期生长、互相链接
- 当你不想让内容生产停留在素材拼接层

## 什么时候用 Content Pipeline
- 当你的目标是产出和复用内容
- 当你需要记录同一选题的不同角度和版本
- 当你需要把发布后的反馈重新喂回系统
