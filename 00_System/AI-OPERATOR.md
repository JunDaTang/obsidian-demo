# AI OPERATOR

> 给未来再次进入这个仓库的 AI / 助手看的。

你下次再进来，大概率会忘记这里的设计意图。所以这个文件不是装饰，而是你的**运行入口**。

## 进入仓库后的默认动作

按顺序执行：
1. 读 `00_System/START-HERE.md`
2. 读 `00_System/WORKFLOW.md`
3. 读 `00_System/VAULT-MAP.md`
4. 读 `00_System/TAG-GLOSSARY.md`
5. 查看 `01_Projects/` 是否有活跃项目
6. 查看 `02_Areas/` 中哪些主题长期活跃
7. 如要产出内容，优先检查：
   - `05_Content/Inbox/`
   - `05_Content/Topic-Cards/`
   - `05_Content/Feedback/`
8. 如要沉淀观点，优先检查：
   - `04_Knowledge/Fleeting/`
   - `04_Knowledge/Literature/`
   - `04_Knowledge/Permanent/`

## 你不能混淆的事情

### 1. Resource ≠ Knowledge
- `03_Resources/` 是原材料仓库
- `04_Knowledge/` 是你已经消化后的认知层

### 2. Knowledge ≠ Content
- 永久卡片不是草稿
- 草稿不是永久卡片
- 内容可以引用永久卡片，但不要反过来把草稿当思想母体

### 3. Project ≠ Area
- `Project` 有开始和结束
- `Area` 是长期经营主题

## 默认工作模式

### 模式 A：知识沉淀
路径：
`Fleeting → Literature → Permanent → MOC`

适用于：
- 阅读、研究、消化外部资料
- 建立自己的观点网络

### 模式 B：内容生产
路径：
`Inbox → Topic → Angle → Draft → Published → Feedback → Repurposing`

适用于：
- 输出文章、短帖、视频脚本、轮播图

### 模式 C：项目执行
路径：
`Project brief → Selected Topics → Drafts → Publishing → Review`

适用于：
- 某月内容实验
- 某次发售预热
- 某主题冲刺

## 你要优先维护的文件
- `00_System/README.md`（如果结构变了）
- `00_System/WORKFLOW.md`（如果流程变了）
- `00_System/TAG-GLOSSARY.md`（如果标签变了）
- `02_Areas/` 下的长期主题概览
- `04_Knowledge/Permanent/` 下的重要认知卡片
- `05_Content/Feedback/` 下的高价值复盘

## 快速判断：一条东西该放哪？
- 如果是摘抄 / 来源材料 → `03_Resources/` 或 `04_Knowledge/Literature/`
- 如果是一闪而过的念头 → `04_Knowledge/Fleeting/`
- 如果是已经成型的长期观点 → `04_Knowledge/Permanent/`
- 如果是可做内容的话题 → `05_Content/Topic-Cards/`
- 如果是同一主题的不同切法 → `05_Content/Angle-Cards/`
- 如果是具体文案 → `05_Content/Draft-Cards/`
- 如果发出去了 → `05_Content/Published/`
- 如果有数据和评论总结 → `05_Content/Feedback/`

## 别犯的典型错误
- 把所有笔记都塞进 Resources
- 把 Draft 当 Permanent 保存
- 发完不写 Feedback
- 没有将高表现内容放进 Repurposing
- 活跃项目不单独管理，导致上下文丢失

## 最终目标
这个仓库不是为了“记很多笔记”，而是为了：
1. 形成可复用的认知资产
2. 支持稳定的内容生产
3. 让未来的人类或 AI 都能接手并持续运作
